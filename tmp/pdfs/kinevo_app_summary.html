Generated from repository evidence only


What it is

Kinevo is a trainer-student fitness platform split into a web app for workout prescription and student management, plus a mobile app for workout execution and logging. It uses Supabase for auth/data and includes Stripe subscription gating in the web product.


Who it is for

Primary persona: strength coaches/personal trainers who prescribe structured programs, with students as execution users in the mobile app.


What it does

• Trainer web dashboard with auth-protected access and student list management.

• Exercise library with system plus trainer-owned exercises and muscle group metadata.

• Program template builder (programs, workouts, ordered workout items including exercise/superset/note types).

• Program assignment flow from templates to student-specific assigned programs/workouts/items.

• Mobile workout player with per-set logging (load, reps, completion), finish flow, and session feedback modal.

• Mobile home/history/profile tabs, including workout history/performance views and support via WhatsApp deep link.

• Stripe subscription checkout + webhook sync; inactive subscriptions are redirected to a blocked page.


How it works (architecture)

• Monorepo workspaces: web (Next.js App Router), mobile (Expo Router/React Native), shared (shared TypeScript types), supabase/migrations (database schema).

• Web data path: Next.js server routes/components call Supabase SSR clients; auth/session protected by middleware; Stripe API routes create checkout/portal sessions and webhook handlers upsert subscription state.

• Mobile data path: Expo app uses Supabase JS with SecureStore-backed auth session persistence; screens/hooks fetch active programs/sessions and submit workout session logs.

• Database model includes trainers, students, exercises, program templates, assigned program snapshots, workout sessions, set logs, and subscriptions with RLS policies.

• Offline resilience signals in schema: sync status, local_id/device_id fields for sessions and set logs.


How to run (minimal getting started)

1. Install deps at repo root: npm install

2. Configure web env in /Users/gustavoprado/kinevo/web/.env.local: NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, STRIPE_SECRET_KEY, STRIPE_PRICE_ID, STRIPE_WEBHOOK_SECRET

3. Configure mobile env in /Users/gustavoprado/kinevo/mobile/.env: EXPO_PUBLIC_SUPABASE_URL, EXPO_PUBLIC_SUPABASE_ANON_KEY

4. Apply Supabase SQL migrations in order from /Users/gustavoprado/kinevo/supabase/migrations to your Supabase project.

5. Run web: npm run web

6. Run mobile: npm run mobile
