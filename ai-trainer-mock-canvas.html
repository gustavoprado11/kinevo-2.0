Por que fecha com o código: a tabela e o card são os mesmos componentes do Kinevo (reuso de ProgramTable/ContractCard), montados pela IA via streamUI. Editar a célula chama a server action / tool real (kinevo_update_workout_item, generate_checkout_link) — então o que aparece no card é o estado, com salvamento otimista. A cobrança continua passando pelo gate confirm. Diff visual (linha verde = adicionada, riscada = removida, roxo = alterado) deixa a IA auditável.
